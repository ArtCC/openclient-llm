Résumé

OpenClient ne collecte, ne stocke, ne transmet ni ne partage aucune donnée personnelle avec le développeur. L'application fonctionne entièrement sur votre appareil et communique uniquement avec le serveur que vous configurez. Aucune analyse, télémétrie ou rapport de crash n'est envoyé au développeur.

Aucun serveur côté développeur

OpenClient ne dispose d'aucun serveur, backend, service d'analyse ou infrastructure cloud exploités par le développeur. Il n'y a aucune télémétrie, aucun suivi d'utilisation et aucun rapport de crash envoyé au développeur. Le développeur ne reçoit jamais aucune donnée liée à votre utilisation de l'application.

Stockage local et iCloud

Toutes les données de l'application sont stockées sur votre appareil ou dans votre compte iCloud personnel. Le développeur n'y a accès à aucune d'elles. En particulier :

  • Historique des conversations — stocké localement dans le répertoire de documents de l'application. Si iCloud Drive est activé, les conversations sont synchronisées entre vos appareils Apple via votre stockage iCloud Drive personnel.
  • Paramètres et profil utilisateur — stockés localement dans UserDefaults. Si iCloud est activé, les données du profil sont synchronisées via iCloud Key-Value Store, privé pour votre identifiant Apple.
  • Éléments de mémoire — stockés localement dans UserDefaults. Si iCloud est activé, les éléments de mémoire sont synchronisés via iCloud Key-Value Store, privé pour votre identifiant Apple. Cela inclut les éléments créés automatiquement par les modèles lors des conversations lorsque des outils de mémoire d'agent sont invoqués.
  • Clés API et URL du serveur — stockées exclusivement dans le Trousseau de l'appareil via le framework Security d'iOS/macOS. Jamais transmises au développeur.
  • Index Spotlight — les titres des conversations et les extraits de messages sont indexés localement dans Spotlight pour permettre la recherche sur l'appareil. Ces données ne quittent pas votre appareil.

Rien ne quitte votre appareil, sauf les requêtes que vous envoyez explicitement au serveur que vous configurez.

Communication avec le serveur

L'application se connecte exclusivement à l'URL du serveur que vous fournissez — une instance Ollama, un proxy LiteLLM ou tout autre backend compatible OpenAI. Toutes les requêtes sont envoyées directement depuis votre appareil vers votre serveur. Cela comprend :

  • Messages de chat et contexte de conversation
  • Images et documents joints aux messages (Vision)
  • Données audio pour la transcription vocale (Speech-to-Text via /v1/audio/transcriptions)
  • Texte pour la synthèse vocale (Text-to-Speech via /v1/audio/speech)
  • Invites pour la génération d'images (/v1/images/generations)
  • Requêtes de recherche web, que votre serveur LiteLLM transmet au fournisseur de recherche que vous avez configuré côté serveur (ex. : Brave, Tavily, DuckDuckGo). L'application ne contacte jamais directement les fournisseurs de recherche.
  • Entrées et sorties des appels d'outils lorsque le mode agent est actif

Le développeur n'a aucun accès, aucune connaissance ni aucune implication dans cette communication.

Contenu partagé et importé

Le contenu reçu via l'extension de partage, le schéma d'URL personnalisé (openclient://), le glisser-déposer ou les Raccourcis Apple est traité entièrement sur votre appareil et envoyé uniquement à votre serveur configuré. Le développeur ne reçoit pas et n'a pas accès à tout contenu partagé.

Aucun SDK tiers ni analyses

OpenClient a zéro dépendance externe pour sa fonctionnalité en temps d'exécution. Il n'y a aucun réseau publicitaire, framework de suivi ni SDK d'analyse intégré dans l'application. Le seul outil tiers utilisé au moment de la compilation est SwiftLint, un outil d'analyse statique du code qui s'exécute lors de la compilation et n'est jamais inclus dans le binaire de l'application publiée.

L'application intègre le SDK Votice — un projet open source du même auteur — exclusivement pour permettre aux utilisateurs de soumettre des rapports de bugs et des demandes de fonctionnalités depuis l'écran Réglages. Les rapports sont entièrement anonymes : aucune identité d'utilisateur, compte, identifiant d'appareil ni donnée personnelle n'est jointe à une soumission. Lorsque vous utilisez cette fonctionnalité, seul le texte que vous saisissez volontairement est envoyé au service Votice. Aucune autre donnée de l'application (conversations, clés API, URL du serveur, mémoire ou pièces jointes) n'est jamais envoyée à Votice. L'utilisation de cette fonctionnalité est entièrement facultative.

Open source et vérifiable

OpenClient est entièrement open source sous la licence Apache 2.0. Vous pouvez inspecter, auditer et vérifier chaque ligne de code dans le dépôt GitHub. Cette transparence vous permet de confirmer qu'il n'existe aucune collecte de données.

Confidentialité des enfants

L'application ne collecte sciemment aucune information de qui que ce soit, y compris les enfants de moins de 13 ans. Puisqu'aucune donnée n'est collectée, il n'y a aucun risque de collecter des données d'enfants.

Modifications de cette politique

Si cette politique est mise à jour, les changements seront publiés sur cette page avec une date de révision mise à jour. Puisque l'application ne collecte aucune donnée, des changements significatifs sont peu probables.
