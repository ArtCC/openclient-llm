Ce que fait l'application

OpenClient est un client Apple natif pour iOS, iPadOS et macOS qui se connecte à tout serveur compatible OpenAI que vous fournissez — tel que Ollama, LiteLLM ou tout autre backend auto-hébergé ou tiers implémentant l'API OpenAI. L'Application permet des conversations textuelles, l'analyse d'images et de documents (Vision), la transcription vocale (Speech-to-Text), la lecture de texte-parole, la génération d'images, la recherche web et une boucle d'appels d'outils agentique — tout cela traité exclusivement par le serveur que vous configurez. L'Application n'héberge, n'exploite ni ne contrôle aucun serveur LLM, modèle d'IA ou fournisseur de recherche.

Les capacités supplémentaires incluent :

  • Synchronisation des conversations : Les conversations sont stockées localement sur votre appareil et, si iCloud est activé, synchronisées entre vos appareils Apple via iCloud Drive.
  • Mémoire : Les éléments de mémoire persistants sont stockés dans iCloud Key-Value Store (lorsque disponible) ou localement dans UserDefaults, et injectés dans les conversations comme contexte. Les modèles peuvent également créer des éléments de mémoire de manière autonome lors d'une conversation.
  • Recherche web : Lorsqu'elle est activée, l'Application envoie des requêtes de recherche via votre serveur LiteLLM au fournisseur de recherche configuré sur ce serveur (ex. : Brave, Tavily, DuckDuckGo). L'Application ne contacte jamais directement les fournisseurs de recherche.
  • Mode agent : L'Application prend en charge une boucle d'appels d'outils agentique dans laquelle le modèle peut invoquer des outils enregistrés et recevoir leurs résultats avant de générer une réponse finale.
  • Extension de partage et Raccourcis : L'Application peut recevoir du contenu d'autres applications via l'extension de partage iOS/iPadOS, un schéma d'URL personnalisé (openclient://), le glisser-déposer ou les Raccourcis Apple.
  • Spotlight : Les titres de conversations et extraits de messages sont indexés localement dans Spotlight pour une recherche rapide. Ces données ne quittent pas votre appareil.

Données locales et iCloud

L'Application stocke l'historique des conversations localement sur votre appareil via le système de fichiers. Si iCloud est activé dans les paramètres de votre appareil, les données suivantes peuvent être synchronisées avec iCloud :

  • Fichiers de conversations (via iCloud Drive)
  • Paramètres du profil utilisateur (via iCloud Key-Value Store)
  • Éléments de mémoire (via iCloud Key-Value Store)

Vous contrôlez si la synchronisation iCloud est active via les paramètres iCloud de votre appareil. Le développeur n'accède à aucune donnée stockée dans votre compte iCloud. Désactiver la synchronisation iCloud signifie que les données restent uniquement locales et ne sont pas partagées entre les appareils.
