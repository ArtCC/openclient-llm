Resumo

O OpenClient não recolhe, armazena, transmite nem partilha quaisquer dados pessoais com o programador. A aplicação funciona inteiramente no seu dispositivo e comunica apenas com o servidor que configurar. Não são enviadas quaisquer análises, telemetria ou relatórios de falhas ao programador.

Sem servidor do programador

O OpenClient não possui servidor, backend, serviço de análise nem infraestrutura na nuvem operados pelo programador. Não existe telemetria, rastreamento de utilização nem relatórios de falhas enviados ao programador. O programador nunca recebe quaisquer dados da sua utilização da aplicação.

Armazenamento local e iCloud

Todos os dados da aplicação são armazenados no seu dispositivo ou na sua conta pessoal do iCloud. O programador não tem acesso a nenhum deles. Especificamente:

  • Histórico de conversas — armazenado localmente no diretório de documentos da aplicação. Se o iCloud Drive estiver ativado, as conversas são sincronizadas entre os seus dispositivos Apple através do seu armazenamento pessoal do iCloud Drive.
  • Definições e perfil do utilizador — armazenados localmente em UserDefaults. Se o iCloud estiver ativado, os dados do perfil são sincronizados através do iCloud Key-Value Store, privado para o seu Apple ID.
  • Itens de memória — armazenados localmente em UserDefaults. Se o iCloud estiver ativado, os itens de memória são sincronizados através do iCloud Key-Value Store, privado para o seu Apple ID. Isto inclui itens criados automaticamente pelos modelos durante as conversas quando as ferramentas de memória do agente são invocadas.
  • Chaves de API e URL do servidor — armazenadas exclusivamente no Keychain do dispositivo através do framework Security do iOS/macOS. Nunca transmitidas ao programador.
  • Índice do Spotlight — os títulos das conversas e excertos de mensagens são indexados localmente no Spotlight para permitir a pesquisa no dispositivo. Estes dados não saem do seu dispositivo.

Nada sai do seu dispositivo, exceto os pedidos que enviar explicitamente ao servidor que configurar.

Comunicação com o servidor

A aplicação liga-se exclusivamente ao URL do servidor que fornece — uma instância Ollama, um proxy LiteLLM ou qualquer outro backend compatível com OpenAI. Todos os pedidos são enviados diretamente do seu dispositivo para o seu servidor. Isto inclui:

  • Mensagens de chat e contexto de conversa
  • Imagens e documentos anexados às mensagens (Vision)
  • Dados de áudio para transcrição de voz (Speech-to-Text via /v1/audio/transcriptions)
  • Texto para síntese de voz (Text-to-Speech via /v1/audio/speech)
  • Prompts para geração de imagens (/v1/images/generations)
  • Consultas de pesquisa na web, que o seu servidor LiteLLM reencaminha para o fornecedor de pesquisa que configurou no servidor (ex.: Brave, Tavily, DuckDuckGo). A aplicação nunca contacta diretamente os fornecedores de pesquisa.
  • Entradas e saídas de chamadas de ferramentas quando o modo agente está ativo

O programador não tem acesso, conhecimento nem envolvimento nesta comunicação.

Conteúdo partilhado e importado

O conteúdo recebido através da Extensão de partilha, do esquema de URL personalizado (openclient://), de arrastar e largar, ou dos Atalhos Apple é processado inteiramente no seu dispositivo e enviado apenas para o servidor configurado. O programador não recebe nem tem acesso a qualquer conteúdo partilhado.

Sem SDKs de terceiros nem análises

O OpenClient tem zero dependências externas para a sua funcionalidade em tempo de execução. Não estão integradas redes publicitárias, frameworks de rastreamento nem SDKs de análise na aplicação. A única ferramenta de terceiros utilizada em tempo de compilação é o SwiftLint, uma ferramenta de análise estática de código que é executada durante a compilação e nunca é incluída no binário da aplicação publicada.

A aplicação integra o SDK Votice — um projeto open source do mesmo autor — exclusivamente para permitir aos utilizadores enviar relatórios de erros e pedidos de funcionalidades a partir do ecrã de Definições. Os relatórios são completamente anónimos: nenhuma identidade de utilizador, conta, identificador de dispositivo nem dados pessoais são anexados a uma submissão. Quando utiliza esta funcionalidade, apenas o texto que introduz voluntariamente é enviado ao serviço Votice. Nenhum outro dado da aplicação (conversas, chaves de API, URL do servidor, memória ou anexos) é alguma vez enviado ao Votice. A utilização desta funcionalidade é completamente opcional.

Open source e auditável

O OpenClient é totalmente open source sob a licença Apache 2.0. Pode inspecionar, auditar e verificar cada linha de código no repositório GitHub. Esta transparência permite-lhe confirmar que não existe qualquer recolha de dados.

Privacidade das crianças

A aplicação não recolhe conscientemente informações de ninguém, incluindo crianças com menos de 13 anos. Uma vez que não são recolhidos quaisquer dados, não existe risco de recolher dados de crianças.

Alterações a esta política

Se esta política for atualizada, as alterações serão publicadas nesta página com uma data de revisão atualizada. Uma vez que a aplicação não recolhe dados, alterações significativas são improváveis.
